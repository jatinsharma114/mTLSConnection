Step 3 — Create localhost.cnf

Ye file SAN (Subject Alternative Name) ke liye hai.

Create:

localhost.cnf bano local machine m

Content:

[req]
default_bits = 2048
prompt = no
default_md = sha256
distinguished_name = dn
req_extensions = req_ext

[dn]
CN = localhost

[req_ext]
subjectAltName = @alt_names

[alt_names]
DNS.1 = localhost
IP.1 = 127.0.0.1
Ye file kyun?

Aaj ke browsers aur Postman CN ko ignore karte hain.

Wo SAN check karte hain.

Without SAN:

SSL Error:

Hostname/IP does not match certificate's altnames

With SAN:

localhost

↓

↓

Step 4 — Generate CSR

Command:

openssl req \
-new \
-key skoda-server.key \
-out skoda-server.csr \
-config localhost.cnf

Generated:

skoda-server.csr
CSR me kya hota hai?

CSR = Certificate Signing Request

Isme hota hai:

Server Public Key
CN = localhost
SAN = localhost
Organization details

Private key (skoda-server.key) CSR me nahi hoti.

Step 5 — Sign the CSR

Yahan tum CA ban ke act kar rahe ho.

Command:

openssl x509 \
-req \
-in skoda-server.csr \
-CA SkodaCA.crt \
-CAkey SkodaCA.key \
-CAcreateserial \
-out skoda-server.crt \
-days 365 \
-sha256 \
-extfile localhost.cnf \
-extensions req_ext

Step 7 — Create PKCS12 Keystore

Spring Boot directly .key aur .crt use nahi karta.

Usko ek PKCS12 Keystore (.p12) chahiye.

Run:

openssl pkcs12 \
-export \
-out skoda-server.p12 \
-inkey skoda-server.key \
-in skoda-server.crt \
-certfile SkodaCA.crt \
-name server \
-password pass:password


Is file ke andar kya hai?
skoda-server.p12

│
├── Private Key
│
├── Server Certificate
│
└── Root CA Certificate


Ek hi file me sab pack ho gaya.

Isi ko kehte hain Keystore.


Step 8 — Verify Keystore

Run:

keytool -list -v \
-keystore skoda-server.p12 \
-storetype PKCS12 \
-storepass password

Step 9 — Create Truststore

Truststore ka purpose:

Main kin Certificates / CAs par trust karta hu?

Create truststore.p12

Run:

keytool \
-importcert \
-alias skoda-ca \
-file SkodaCA.crt \
-keystore truststore.p12 \
-storetype PKCS12 \
-storepass password \
-noprompt

--------------

Verify Truststore

Run:

keytool -list -v \
-keystore truststore.p12 \
-storetype PKCS12 \
-storepass password
